# 🧭 The 10-Minute Figma → React Method

## 1) Scan the screen (don’t code yet)

Open your file in Figma and identify **sections**, not frames:

```
Hero
 ├── Navbar
 ├── Title (categories + heading)
 ├── Search Bar
 └── Cards (Most Popular)
```

👉 These become your **top-level components**.

---

## 2) Draw the component tree (2–3 mins)

Write it like this (on paper or notes):

```txt
HeroSection
 ├── Navbar
 ├── TitleSection
 ├── SearchBar
 └── CardsSection
      ├── SectionHeader
      └── CardList
           └── DestinationCard
```

👉 If something repeats → make it a component (Card, Button, Input).

---

## 3) Map Auto Layout → Flex/Grid

In Figma Inspect:

* Auto layout (Horizontal) → `flex`
* Auto layout (Vertical) → `flex-col`
* Spacing → `gap`
* Padding → `p-*`

Example:

```jsx
// Figma: horizontal auto layout, gap 16
<div className="flex items-center gap-4">
```

---

## 4) Build the skeleton first (no styles)

Create files and return bare JSX:

```jsx
// Home.jsx
export default function Home() {
  return (
    <HeroSection>
      <Navbar />
      <TitleSection />
      <SearchBar />
      <CardsSection />
    </HeroSection>
  );
}
```

👉 Don’t style yet. Just structure.

---

## 5) Add layout styles (containers first)

Start from the outside:

```jsx
// HeroSection.jsx
export default function HeroSection({ children }) {
  return (
    <section className="relative min-h-screen">
      <div className="max-w-7xl mx-auto px-4 sm:px-6 md:px-10">
        {children}
      </div>
    </section>
  );
}
```

👉 Always use:

* `max-w-*` (center content)
* `px-*` (responsive padding)

---

## 6) Convert one component at a time

### Example: Search Bar (from your Frame 30)

Figma:

```
Location | Check-in | Check-out | Guests | Button
```

React:

```jsx
export default function SearchBar() {
  return (
    <div className="mt-10 bg-white/50 backdrop-blur-lg rounded-xl p-4">
      <div className="flex flex-col md:flex-row md:items-center gap-4 md:gap-6">

        <div className="flex-1">
          <p className="text-xs text-gray-500">Location</p>
          <p className="font-semibold">Where are you going?</p>
        </div>

        <div className="hidden md:block w-px h-10 bg-gray-300" />

        <div className="flex-1">
          <p className="text-xs text-gray-500">Check-in</p>
          <p className="font-semibold">15 Sep 2026</p>
        </div>

        <button className="bg-[#E7AC72] text-white px-6 py-3 rounded-xl">
          Search
        </button>

      </div>
    </div>
  );
}
```

👉 This is a **direct mapping** of Figma → React.

---

## 7) Handle responsiveness early

Use mobile-first classes:

```jsx
<div className="
  flex flex-col gap-4
  md:flex-row md:items-center
">
```

👉 Rule:

* mobile = stacked
* desktop = row/grid

---

## 8) Convert cards (repeatable UI)

Figma shows multiple cards → build once:

```jsx
function DestinationCard({ image, title, people }) {
  return (
    <div className="bg-white rounded-xl shadow p-2">
      <img src={image} className="h-36 w-full object-cover rounded-md" />
      <h3 className="text-sm font-semibold mt-2">{title}</h3>
      <p className="text-xs text-gray-500">{people}</p>
    </div>
  );
}
```

Then list:

```jsx
<CardList data={data} />
```

---

## 9) Add polish (last 10%)

Only now:

* shadows
* hover effects
* gradients
* animations

Example:

```jsx
className="hover:scale-105 transition duration-300"
```

---

## 10) Final check (this separates pros)

Ask yourself:

✔ No fixed widths like `w-[400px]`
✔ No unnecessary nested divs
✔ Components reusable
✔ Works on mobile

---

# 🧠 Real Developer Mental Model

> Figma shows **how it looks**
> React defines **how it’s built**

# 🔥 Quick Cheat Sheet

| Figma       | React        |

| Auto Layout | flex         |
| Spacing     | gap          |
| Padding     | p-*          |
| Color       | bg-*         |
| Text        | text-*       |
| Component   | reusable JSX |
